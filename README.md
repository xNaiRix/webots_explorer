# Webots Explorer - Автономный робот-разведчик

Финальный проект по курсу "Мехатроника". Автономный робот на базе e-puck в симуляторе Webots, выполняющий миссию разведки: исследует комнату с препятствиями, фиксирует информацию о среде и возвращается на стартовую позицию.

## Описание проекта

Проект реализует автономного робота-разведчика, который:
1. Стартует из точки (0, 0) в симулированной среде Webots
2. Исследует комнату, объезжая препятствия с использованием датчиков расстояния
3. Строит карту окружающего пространства в реальном времени
4. Через заданное время (или после обнаружения определенного числа препятствий) переходит в режим возврата
5. Возвращается к стартовой позиции с использованием одометрии и построенного маршрута
6. Останавливается при достижении стартовой позиции

## Функциональные возможности

### Основные функции
- **Автономная навигация**: движение с избеганием препятствий
- **Картографирование**: построение сеточной карты исследованной области
- **Одометрия**: отслеживание положения и ориентации робота
- **Конечный автомат (FSM)**: управление состояниями робота (исследование, поворот, движение вперед, возврат)
- **Планирование маршрута**: алгоритм поиска пути обратно к стартовой точке
- **Логирование**: запись статистики миссии в файл

### Дополнительные возможности
- Визуализация траектории движения
- Статистика миссии (пройденный путь, время, количество препятствий)
- Настройка параметров через конфигурационные константы
- Расширяемая архитектура состояний

## Архитектура системы

### Компоненты системы

```
┌─────────────────────────────────────────────────────────────┐
│                      RobotController                         │
├─────────────┬──────────────┬──────────────┬─────────────────┤
│   FSM       │   Odometry   │     Map      │    Devices      │
│   Handler   │              │              │                 │
├─────────────┼──────────────┼──────────────┼─────────────────┤
│   States    │   Sensors    │   Router     │   Triggers      │
│   (Forward, │   Processing │   (Path      │   (Schmidt)     │
│   Turn)     │              │   Planning)  │                 │
└─────────────┴──────────────┴──────────────┴─────────────────┘
```

### Конечный автомат (FSM)
Система использует конечный автомат со следующими состояниями:
- **ForwardState**: движение вперед к целевой точке
- **TurnLeftState**: поворот налево на заданный угол
- **TurnRightState**: поворот направо на заданный угол
- **ExploreState**: исследование неизвестной области (реализовано через комбинацию Forward и Turn)
- **ReturnHomeState**: возврат к стартовой позиции

### Модули
1. **RobotController** (`explorer.py`) - главный контроллер, координирующий все компоненты
2. **FSM** (`FSM/`) - реализация конечного автомата и состояний
3. **Odometry** (`odometry.py`) - расчет положения робота на основе энкодеров
4. **Map** (`map.py`) - представление карты и операций с ней
5. **Devices** (`robot/devices.py`) - абстракции датчиков и актуаторов
6. **Router** (`FSM/router.py`) - планировщик маршрутов на карте
7. **Logger** (`logger.py`) - запись статистики и данных миссии

## Установка и запуск

### Предварительные требования
- [Webots R2023b или новее](https://cyberbotics.com/)
- Python 3.8+
- Установленные зависимости (см. `requirements.txt`)

### Установка зависимостей
```bash
pip install -r requirements.txt
```

### Запуск симуляции
1. Откройте Webots
2. Загрузите мир: `worlds/my_scene.wbt`
3. Установите контроллер для робота e-puck:
   - Выберите робота e-puck в сцене
   - В свойствах установите контроллер "explorer"
   - Убедитесь, что путь `controllers/explorer` указан правильно
4. Запустите симуляцию

### Альтернативный запуск через командную строку
```bash
webots --mode=run worlds/my_scene.wbt
```

## Использование

### Основной цикл работы
1. Робот инициализируется в центре координат (100, 100 в клетках карты)
2. Начинается фаза исследования:
   - Робот движется вперед, пока не обнаружит препятствие
   - При обнаружении препятствия выполняет поворот
   - Обновляет карту исследованной области
   - Едет к "ближайшей" неисследованной области
   - Ищет ближайшую неисследованную область с помощью алгоритма Дейкстрв
3. После достижения лимита времени или обнаружения N препятствий:
   - Активируется режим возврата
   - Планировщик строит маршрут к стартовой точке
   - Робот следует по маршруту, объезжая известные препятствия
4. При полном исследовании доступной территории робот перехоодит в состояние возврата на старт
5. При достижении старта

### Настройка параметров
Основные параметры настраиваются в `controllers/explorer/constants.py`:
- `CELL_SIDE = 25` - размер стороны клетки карты в мм
- `ROBOT_WHEEL_BASE = 52` - база колес робота в мм
- `ROBOT_WHEEL_RADIUS = 20.5` - радиус колес в мм
- `MAX_COORD_ERROR = 5` - максимальная ошибка позиционирования в мм

### Логирование
Логгер записывает информацию о траектории движения робота и показаниях датчиков в реальном времени. Данные сохраняются для последующего анализа и визуализации.

## Визуализация данных

Проект включает скрипты для визуализации данных миссии:

### Визуализация траектории
```bash
cd statistics/scripts
python visualize_trajectory.py
```
Генерирует график пути робота в координатах X-Y.

### Визуализация карты
```bash
python visualize_map.py
```
Показывает исследованную область и препятствия.

### Визуализация показаний датчиков
```bash
python visualize_sensors_stats.py
```
Отображает историю показаний датчиков расстояния.

## Требования

### Аппаратные (симулированные)
- Робот e-puck с 8 датчиками расстояния
- Среда с препятствиями (стены, объекты)

### Программные
- **Webots**: R2023b или новее
- **Python**: 3.8+
- **Библиотеки Python**:
  - `matplotlib>=3.5.0` (для визуализации)
  - `numpy>=1.21.0` (для вычислений)
  - Стандартные библиотеки: `math`, `sys`, `os`, `typing`

### Системные
- ОС: Windows, Linux или macOS
- 4 ГБ ОЗУ (минимум)
- Графическая карта с поддержкой OpenGL
